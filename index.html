1. 문서 개요

요약

이 페이지는 사용자(Front) PDF 화면설계서와 관리자(CMS) PPTX 화면설계서를 기반으로, 인수인계 목적에 맞게 핵심 기능/규칙/연동/상태값 중심으로 재구성한 요약본입니다. “처음 보는 사람도” 프로젝트를 따라갈 수 있도록 IA → 연동 → 공통룰 → (사용자) 모듈별 → (관리자) 메뉴별 순서로 정리했습니다.

요건
  • 최종 근거는 항상 원본 문서(화면설계서)로 확인(문구/레이아웃/예외 포함).
  • 좌측 검색(Ctrl+K)으로 화면ID/상태값/키워드 기반 탐색.
  • 각 섹션은 “요약 → 요건(필수 규칙) → 상세” 순서로 읽기.

이 페이지는 원본 PDF 화면설계서를 기반으로, 인수인계 목적에 맞게 핵심 기능/규칙/연동/상태값 중심으로 재구성한 요약본입니다. “처음 보는 사람도” 프로젝트를 따라갈 수 있도록 IA → 연동 → 공통룰 → 모듈별 상세 순서로 정리했습니다.

2. IA / 화면 체계

요약

사용자(Front) IA와 화면 코드 체계(UI-*)를 정리하고, 관리자(CMS) 메뉴 구조와의 연결 지점을 함께 표시합니다.

요건
  • Front IA는 Depth1/Depth2 기준으로 페이지/기능 단위를 식별.
  • CMS 메뉴는 '공통관리/프로그램/회원/게시글/통계/시스템' 축으로 관리 포인트를 식별.
  • 동일 개체(전시/교육/시설/회원)는 Front 화면과 CMS 메뉴를 1:1로 매핑해 운영 혼선을 방지.

원본은 화면ID 체계를 사용합니다(예: UI-MV-01-002). 인수인계/협업 시 “화면ID”로 커뮤니케이션하면 가장 빠릅니다.

3. 연동/외부 시스템

요약

Front·CMS 모두에서 사용되는 외부 연동(예매/영화인 검색/알림/관리 API)을 한 곳에 모아, 개발·운영 시 의존성을 빠르게 파악합니다.

요건
  • 디트릭스: 영화 예매/좌석/결제/취소 등 핵심 트랜잭션 연동(오류·지연 대응 정책 필요).
  • KOBIS: 영화인 멤버십 신청 시 영화인 조회/선택(검색 UX/쿼리 제한 고려).
  • CMS: 'API 연계 관리'에서 Endpoint/Key/파라미터 등을 관리할 수 있는 구조(권한·보안 필수).

4. 공통 정책/룰

요약

상태값, 업로드 정책, 보안/제한 등 전 모듈에 영향을 주는 공통 규칙을 정리합니다.

요건
  • 상태값(신청/반려/승인/취소/환불 등)은 Front 표시와 CMS 처리 상태가 일치해야 함.
  • 파일 업로드: 용량/개수/확장자 제한을 공통 컴포넌트로 강제(Front·CMS 정책 분리 주의).
  • 로그인 보안/제한(잠김/해제, 인증 등)은 운영 정책(잠김 기준/해제 절차)을 함께 정의.
상태값(프로세스) 공통 축

문서상 여러 신청/결제 흐름이 공통적으로 아래 패턴을 가집니다.

  • 신청/접수 → 승인/반려(또는 선정/미선정) → 입금/결제 안내 → 입금확인 → 완료
  • “입금확인완료”까지는 시설대관/영화인 멤버십이 동일 프로세스를 공유한다는 설명이 존재합니다.
파일 업로드 정책(대표)
구분 허용 형식 제한
교육 신청내역 수정(첨부) hwp, pdf, doc, docx 단일 업로드, 30MB 이하, 필수 첨부
시설대관 신청(대관신청서) hwp, pdf, doc, docx, xls, xlsx, ppt, pptx 단일 업로드, 30MB 이하
영화인 멤버십 신청(증빙 등) pdf, jpg/jpeg, png, svg 단일 업로드, 30MB 이하

※ 업로드 정책은 운영/보안(웹취약점 진단)과도 연관되므로, 실제 구현 시 서버 측 검증(MIME/확장자/바이러스 스캔 등) 기준을 별도로 확정하세요.

참고(CMS): 메인배너/콘텐츠 업로드는 최대 50MB(1개, jpg/png/jpeg), 게시글 첨부는 최대 50MB·10개까지 허용으로 표기되어 있어 Front(30MB)와 정책 불일치 가능성이 있습니다. 운영/보안 기준으로 최종 통일 필요.

로그인 보안/제한
  • 일반 로그인에서 로그인 정보 5회 불일치 시 시스템 확인창 추가(감리/웹취약점 대응 이력에 포함).
  • 소셜 로그인은 “홈페이지 가입 여부를 판단할 수 없음” 이슈로 별도 안내/연동 플로우를 둠.

5. 회원/인증

요약

회원가입(일반/어린이), 소셜로그인, 아이디/비밀번호 찾기 등 인증/계정 기반 기능을 다룹니다.

요건
  • 일반/어린이 가입 분기 및 필수 정보(약관/본인확인 등) 흐름을 화면 기준으로 구현.
  • 네이버/카카오 소셜 연동: 가입/로그인/연동해제(마이페이지)까지 일관된 정책 필요.
  • 계정 잠김/비밀번호 초기화는 CMS의 '비밀번호 초기화/잠김해제' 기능과 연계.
회원가입(일반/어린이) 핵심 포인트
  • 일반/어린이 가입 플로우가 분리되어 있으며, 본인인증 수단/필드가 개정 이력에서 여러 번 변경되었습니다(이메일 ↔ 아이핀 등).
  • 회원가입 시 선호장르 정보를 활용해 영화 영역에서 “이 영화 어때요?” 추천 아이콘에 영향을 줍니다.
소셜 로그인(네이버/카카오)
  • 소셜 제공 데이터만으로 “이미 홈페이지 회원인지” 판단이 어렵다는 사유로, 미가입/미연동 확인창을 둡니다.
  • 네이버는 “네이버 식별자”로 기존 가입자 여부를 판단하는 흐름이 설계되어 있습니다.
  • 마이페이지에서 소셜 계정 연동/해제 가능(연동 시 연결 상태 표시, 해제 시 확인창).
Kakao Naver 연동/미가입 분기
아이디/비밀번호 찾기
  • 아이디/비밀번호 찾기 인증 수단이 분리(휴대폰 인증/이메일 인증), 비밀번호 찾기에서 이메일 본인인증 수단 추가 등 이력이 존재합니다.
  • 실제 구현 시 “어떤 인증수단이 최종 확정인지”는 운영/정책 문서(또는 개발 완료 버전)와 대조 필요합니다.

6. 영화 예매/확인

요약

영화 목록/상세/상영시간표/예매확인 등 '영화 관람' 핵심 여정을 정리합니다(디트릭스 연동 중심).

요건
  • 예매/취소/좌석/결제 등 상태는 디트릭스 응답과 동기화되어야 함.
  • 추천/태그/노출 규칙은 운영자가 CMS에서 관리 가능한 범위를 명확히 해야 함.
  • 장애/지연 시 사용자 안내(재시도/문의)와 운영 모니터링(오류 로그) 준비.
기능 범위
  • 상영예매(상영중/상영예정) 목록, 상세
  • 인원/좌석 선택, 할인 선택, 결제
  • 예매내역 확인(회원), 비회원 예매 및 확인
  • 지난 상영작 목록/상세(아카이브 성격)
  • 할인혜택 안내 페이지(운영 규정 반영)
디트릭스 연동 관리자 설정(예고편)
디트릭스 연동 포인트(요약)
  • 영화 정보(제목/개봉일/장르/러닝타임 등)를 디트릭스 API로 불러오는 설계가 포함됩니다.
  • 영화 예매는 “디트릭스 영화예매 솔루션 연동”으로 명시되어 있습니다.
  • 할인 선택은 디트릭스에서 할인 종류 정보를 불러오며, 조건에 따라 “할인 적용이 불가합니다” 등의 안내가 노출됩니다.
추천/태그
  • “이 영화 어때요?” 아이콘: 회원가입 시 등록된 선호장르에 해당하는 영화일 경우 표시.
  • 상영 상태 태그 예: 상영예정(D-남은일수), 상영중, 상영마감 등.
운영/확인 포인트
  • 영화 상세에서 “메인 예고편 영상”은 관리자에서 설정한 영상 불러오기(기능 확인 필요)로 명시되어 있습니다.
  • 예매 취소 가능 시간/극장별 유의사항 등은 예매내역에서 표기합니다.

7. 전시·행사

요약

전시·행사 콘텐츠의 목록/상세/신청 및 운영 포인트를 정리합니다.

요건
  • 전시/행사 기본정보(기간/시간/장소/문의/가격 등)는 CMS 등록 항목과 동일한 스키마로 유지.
  • 신청/참여가 있는 경우, CMS에서 신청자 현황/처리 상태가 조회 가능해야 함.
  • 휴관일/미노출 등 운영 변수는 캘린더·상세 노출과 충돌하지 않도록 정책 확정.
기능 범위
  • 전시·행사 목록/상세(진행상태 태그 포함)
  • 상세 상단 비고 항목, 공유 기능(링크 복사 등) 설계가 포함됩니다.
  • 진행상태 워딩 개정 이력: “진행종료 → 종료”
운영 포인트
  • 콘텐츠(전시/행사 정보)는 관리자에서 관리되는 전형적인 게시형 구조로 보입니다.
  • 통합캘린더/통합검색에도 노출되므로, 노출 필드(제목/기간/장소/썸네일/태그) 일관성 중요.

8. 교육

요약

교육 프로그램의 신청/결제(해당 시)/취소·환불/첨부 등 사용자 흐름과 상태를 정리합니다.

요건
  • 교육 유형/정원/대기/마감 등 상태 규칙을 Front와 CMS가 동일하게 사용.
  • 취소·환불 규정(기한/수수료/불가 조건)은 사용자 안내 문구까지 포함해 확정.
  • CMS에서 신청자 현황, 설문, 수료증 발급 등 운영 기능과 데이터 구조를 맞춤.
핵심 흐름
  1. 교육강좌 목록/상세에서 신청
  2. 신청 완료 화면(유형별 안내 문구 상이)
  3. 결제는 “접수 마감 후” 진행(마이페이지 신청내역에서 결제)
  4. 입금 확인/상태 변경(관리자 처리 + 일부 자동 변경)
  5. 필요 시 취소/환불(환불계좌 등록 후 환불요청)
무통장입금(계좌) 알림톡
유형(개정 이력 반영)
  • 기존: 성인/어린이/청소년 → 변경: 영화인 교육(영화인 대상), 일반 교육(전체/성인/어린이·청소년)
결제/쿠폰
  • 마이페이지 결제 화면에서 쿠폰 드롭다운으로 미사용 강좌 할인 쿠폰을 선택(없으면 “사용 가능한 쿠폰이 없습니다”).
  • 입금계좌: 서울영화센터 계좌 표시, 입금자명: 회원가입 시 등록된 이름.
  • 결제하기 완료 후 결제 안내 알림톡 발송(어린이 교육은 신청자+자녀 번호).
상태/취소·환불
  • 마이페이지 취소/환불내역 탭(교육/시설대관) 제공.
  • 교육 상태 필터 예시: 대기신청취소, 신청취소, 수강취소, 환불요청, 환불완료.
  • 환불요청 버튼 클릭 시 환불요청 확인창 → 환불계좌 등록 모달 등 흐름이 설계되어 있습니다.
첨부파일
  • 교육 신청/수정 시 첨부파일 업로드가 포함됩니다(형식/용량 제한 존재).

9. 시설대관

요약

시설대관(공유오피스/상영관 등) 신청·예약·운영(입금확인/이용완료 등)을 정리합니다.

요건
  • 예약(날짜/시간/좌석/목적/요청사항) 입력값 검증 및 캘린더 UI의 오류 방지.
  • 입금확인 등 운영 상태 변경은 CMS에서 처리하며, Front 조회 상태와 일치 필요.
  • 대관신청서 등 첨부파일 정책(확장자/용량)과 보관/다운로드 권한을 확정.
기능 범위
  • 시설대관 목록/상세(시설별 안내/대관료/신청서 양식)
  • 예약 캘린더/시간 선택, (상영관은 좌석 선택 포함)
  • 대관 신청(파일 첨부), 입금/승인/완료 상태 관리
무통장입금(계좌) 기능 확인 필요(문서 표기)
예약(캘린더/시간/좌석) 핵심
  • 예약폼 상세 기능 확인(문서 내 명시).
  • 시설 유형(상영관/다목적실/회의실/공유오피스)별로 최소 예약 시간 단위 및 날짜 선택 범위가 다릅니다.
  • 공유오피스는 영화인 멤버십 회원만 예약 가능이라고 명시되어 있습니다.
  • 상영관은 좌석 선택: 이미 예약된 좌석은 선택 불가, 좌석 선택은 필수.
결제/운영
  • 시설대관 신청 후 입금 확인 프로세스가 있으며, 계약금/잔금 등은 운영에서 관리한다는 설명이 포함됩니다.
  • 시설대관과 영화인 멤버십은 “입금확인완료”까지 공통 프로세스를 공유합니다.
첨부파일(대관신청서)
  • 신청서 다운로드 후 작성 → 파일 첨부 업로드
  • 형식: hwp, pdf, doc, docx, xls, xlsx, ppt, pptx / 30MB 이하 / 단일 업로드

10. 멤버십

요약

일반 멤버십(등급 산정/혜택/쿠폰)의 운영 규칙을 정리합니다.

요건
  • 등급 산정 기준(홈페이지 예매+관람완료, 현장예매 제외)과 산정 시점을 시스템화.
  • 쿠폰 혜택은 발급/노출/사용 집계 정책을 명확히(특히 오프라인 사용 처리).
  • 멤버십 통계(등급/신청/쿠폰 사용 수)는 CMS 통계 메뉴와 지표 정의가 필요.
일반 멤버십(등급/혜택)
  • 등급 예시: 현무, 주작, 청룡, 백호
  • 등급 산정: 1년(1~12월) 영화 관람 실적 기준(홈페이지를 통해 예매 + 관람완료 건만 집계, 현장예매 제외).
  • 2026년은 예외적으로 상반기(1~6월) 관람실적으로 일괄 적용한다는 설명이 포함됩니다.
  • 등급 산정 시점: 매년 1월, 등급 유지 기간: 1년.
쿠폰/할인(운영)
  • 멤버십 혜택으로 쿠폰이 발급/노출될 수 있으며, 식음료 할인쿠폰의 경우 “매점에서 쿠폰 제시 → 직원이 할인 적용 확인 후 적용” 형태의 흐름이 설계되어 있습니다.
  • 정확한 혜택/쿠폰 종류/발급 정책은 “홈페이지 약관 운영 규정 반영” 파일의 해당 시트(문서에서 참조)로 최종 확인 필요.

11. 영화인 멤버십

요약

영화인 멤버십(연회비·무통장입금·KOBIS 영화인 검색 연동)의 신청/상태/환불 규정을 정리합니다.

요건
  • 신청 시 KOBIS 영화인 정보 조회/선택 + 관심 영화 정보(필수) 입력.
  • 상태값(신청/입금확인/확정/반려/환불 등)은 CMS 처리 프로세스와 1:1 대응.
  • 환불 가능 기한(입금확인완료 처리일 기준 7일 등)과 첨부파일 정책을 엄격히 적용.
개요
  • 연회비 60,000원 결제(무통장입금)로 가입하는 별도 멤버십입니다.
  • 신청 시 KOBIS 영화인 검색을 통해 영화인 정보를 조회/선택합니다.
  • 관심 영화 정보(유형/제작국가/장르)를 선택합니다(필수).
KOBIS 연동 무통장입금(계좌)
상태값(대표)

문서에 언급된 상태값/흐름을 인수인계 관점으로 정리했습니다.

  • 신청 → (관리자) 입금확인중 → 입금확인완료
  • 신청확정 / 신청반려 (상태 변경 시 알림톡 발송)
  • 취소/환불요청 → 취소/환불완료
  • 유효기간만료(유효기간 1년)
환불 규정(핵심)
  • 환불요청은 입금확인완료 처리일 기준 7일 이내 가능.
  • 7일 경과 시 환불 요청 불가로 안내됩니다.
신청/수정과 첨부파일
  • 신청/신청반려 상태일 경우 영화인 정보 및 관심 영화 정보 수정 가능.
  • 첨부파일: pdf, jpg/jpeg, png, svg / 30MB 이하 / 단일 업로드.

12. 서울영화센터(정보/고객센터)

요약

센터 소개/이용안내/고객센터(Q&A 등) 성격의 정보성 콘텐츠를 정리합니다.

요건
  • 다국어 페이지(소개)의 번역 범위/콘텐츠 소스(정적/관리자 입력)를 확정.
  • Q&A(1:1 문의) 개인정보 처리/보관/답변 SLA를 운영 정책으로 정의.
  • 공지/이벤트/FAQ 등 게시물은 CMS 게시글 관리와 동일한 필드/첨부 정책을 공유.
구성
  • 서울영화센터 소개(다국어)
  • 공지사항
  • 이벤트(탭: 기타 추가 이력)
  • FAQ(탭: 시설대관 추가 이력)
  • Q&A(모든 업로드 글 비밀글 처리)
다국어(소개 페이지)
  • 소개 페이지에서 다국어 제공(번역 품질/정확도 이슈로 Google Translate 제외 언어에 notranslate 처리 등의 설명이 포함됨).
Q&A(1:1 문의) 정책
  • 모든 업로드 글은 비밀글로 처리된다는 개정 이력이 있습니다.
  • 작성/상세/답변 구조(관리자 답변 포함)를 갖는 전형적인 1:1 문의 게시판 흐름입니다.

13. 통합캘린더

요약

영화/전시/교육/시설 등 여러 일정을 한 화면에서 보는 통합 캘린더 기능을 정리합니다.

요건
  • 캘린더에 포함되는 대상(카테고리)과 필터 규칙을 확정.
  • 휴관일/미노출 콘텐츠가 캘린더에 노출되지 않도록 데이터 필터링 필요.
  • 대량 데이터(월간 조회) 성능과 API 호출량(캐시)을 고려.
무엇을 보여주나?
  • 서울영화센터에서 진행하는 영화 · 전시·행사 · 교육 일정 통합 조회
  • 영화 일정은 실시간으로 디트릭스 API에서 정보 불러옴
운영 포인트
  • 전시·행사/교육은 관리자 데이터 정합성이 중요(캘린더 노출 필드/기간/장소 등).
  • 영화는 외부 연동(디트릭스)이므로 “타임존/상영시간 변경/상영취소” 같은 이벤트 처리 전략 확인 필요.

14. 통합검색

요약

사이트 전체에서 영화/전시/교육/게시물 등 콘텐츠를 찾는 통합검색 범위를 정리합니다.

요건
  • 검색 대상 카테고리/정렬/노출 규칙(하이라이트, 페이징)을 명확히.
  • 개인정보/민감 정보가 검색 결과에 노출되지 않도록 인덱싱 정책 필요.
  • CMS 콘텐츠 변경 시 검색 인덱스 반영(실시간/배치) 방식을 확정.
검색 대상(카테고리)
  • 영화
  • 전시·행사
  • 교육
  • 시설대관
  • 서울영화센터 (공지/이벤트/FAQ/Q&A 등 서브 메뉴 포함)
확인 필요
  • 문서에 “영화예매는 확인필요(디트릭스 API로 바로 불러오는지)”라는 메모가 있습니다.
  • 따라서 검색에서 ‘영화’ 결과가 “콘텐츠(영화 정보)”인지, “상영/예매 영역”까지 포함하는지, 실제 연동 범위를 확정해야 합니다.

15. 마이페이지

요약

마이페이지에서 예약/신청/환불/쿠폰/소셜 연동 등 개인화 기능을 제공하는 범위를 정리합니다.

요건
  • 사용자 신청/예약 내역은 각 도메인(교육/시설/멤버십/영화 예매) 상태와 동기화.
  • 취소/환불은 정책에 따라 버튼 활성/비활성 및 안내 문구가 바뀌도록 구현.
  • 소셜 계정 연동은 로그인 정책과 일관되게(연동해제 시 로그인 수단 확인).
역할
  • 사용자 개인 정보/연동 관리 + 신청/결제/환불의 “업무 허브”
  • 영화 예매 내역(디트릭스 연동), 교육/시설대관 신청내역, 영화인 멤버십 신청/상태
신청내역(교육 예시)
  • 상태별로 결제하기/환불요청/수료증/설문 등의 버튼 노출이 달라집니다.
  • 신청/신청반려 상태에서만 신청정보 수정 가능(수정 시 상태값 ‘신청’으로 변경).
  • 첨부파일 업로드(단일/30MB/형식 제한) 포함.
취소/환불 내역
  • 교육/시설대관 탭 제공, 키워드 검색 및 상태 필터 제공.
  • 환불요청→환불완료는 사용자 요청 + 관리자 처리 단계가 존재합니다.
소셜 계정 연동
  • 카카오/네이버 연동 여부 표시
  • 연동/해제 시 확인창, 해제 처리 후 연동 상태 갱신

16. 관리자(CMS) 개요

요약

관리자(CMS)는 홈페이지 운영자가 배너/팝업/콘텐츠, 프로그램(영화·전시·교육·시설), 회원/멤버십, 통계, 시스템 설정을 관리하는 백오피스입니다.

요건
  • 로그인 후 인증번호 입력 단계가 있어 보안(2FA) 흐름을 구현 방식까지 확정.
  • 알림톡(카카오) on/off 등 운영 설정은 권한 통제 및 로그(누가 언제 변경) 필요.
  • 대시보드 지표는 Front의 집계 기준과 불일치하지 않도록 정의서로 고정.
관리자 시스템 범위
  • 서울영화센터 통합관리시스템(CMS)으로, 홈페이지에 노출되는 콘텐츠/프로그램/회원/통계/시스템 설정을 운영자가 관리합니다.
  • 원본 근거: 서울영화센터 홈페이지 구축 관리자_v0.9.pptx (제출용, v0.9, 2025-09-09 표기)
CMS 운영/관리자 권한/보안 중요
로그인/인증 흐름(2단계)
  • 로그인 페이지: 아이디/비밀번호 입력 + 아이디 기억하기 옵션
  • 추가 인증: 인증번호 입력 단계가 존재(운영 환경에서 2차 인증/OTP/메일·문자 인증 등 구현 방식 확정 필요)
대시보드(Analytics dashboard) 개요
  • 지표 예시: 전체 가입자 인원수, 일반 멤버십 인원수, 영화 멤버십 인원수
  • 운영 포인트: 지표 정의(집계 기준/갱신 주기/필터)를 Front 정책과 일치시켜야 혼선이 없습니다.
관리자 프로필/회원정보수정
  • 관리자 계정 정보(아이디/가입일) 확인
  • 비밀번호 변경(비밀번호/재입력), 이메일/연락처/직급/메모 입력
  • 권한은 “출력” 형태로 노출(권한 변경은 별도 관리자 관리/권한설정 메뉴로 예상)
관리자 알림 설정(카카오 알림톡)
  • 알림채널: 카카오 알림톡
  • 트리거 예시: 영화 등록, 전시행사 등록, 교육목록 등록, 신청자 등록, 시설대관 신청(메뉴별), 회원 가입/탈퇴, 멤버십 신청, FAQ 등록
  • 일괄 on/off 옵션 존재

17. 관리자(CMS) 공통관리(배너/팝업/콘텐츠)

요약

Front 첫 화면에 직접 영향을 주는 메인배너/팝업과 공통 콘텐츠 리소스를 관리합니다.

요건
  • 메인배너: 이미지 규격/용량/확장자 제한 준수(문서 표기 1920×947, 50MB, 1개, jpg/png/jpeg).
  • 팝업: 자동/수동·레이어·위치·사이즈·링크 등 노출 정책을 운영 규칙으로 확정.
  • 콘텐츠 관리: 파일 교체 시 캐시/반영 타이밍과 변경이력(감사 로그) 확보.
메인배너 관리(공통관리 > 메인배너 관리)
  • 기본 입력: 배너명, 태그내용, 사용여부, 링크 URL, 링크 열기(현재창 등)
  • 업로드 정책(문서 표기): 최대 50MB, 1개, jpg/png/jpeg, 이미지 사이즈 1920×947
  • 운영 포인트: 배너 “사용” 값에 따라 Front 노출/비노출이 결정되도록 구현
팝업 관리(공통관리 > 팝업 관리)
  • 기본 입력: 팝업명, 사용여부
  • 표시 방식: 레이어(레이어/팝업 형태), 자동/수동(노출 제어) 항목 존재
  • 위치/사이즈: 가로×세로 입력 항목 존재(레이어 팝업 배치)
  • 콘텐츠: 에디터 영역(본문 HTML) + 링크 URL + 링크 열기(현재창 등)

참고: 원본 문서에 ‘권장 사이즈: 1920×400’ 표기가 있으며(배너/팝업/콘텐츠 중 적용 대상은 화면 기준으로 재확인 필요), 이미지 규격은 운영 가이드로 고정하는 것을 권장합니다.

콘텐츠 관리(공통관리 > 콘텐츠 관리)
  • 목적: 공통 이미지/파일 등 “콘텐츠 리소스”를 업로드·교체하는 영역으로 보입니다.
  • 기능: 콘텐츠 선택, 가져오기 기능, 파일 선택 업로드, 미리보기(진행률), 변경 적용
  • 업로드 정책(문서 표기): 최대 50MB, 1개, jpg/png/jpeg, 이미지 사이즈 “협의” 표기

18. 관리자(CMS) 프로그램/콘텐츠 운영(영화·전시·교육·시설)

요약

영화/전시·행사/교육/시설대관 콘텐츠를 등록하고, 신청자·예약 상태를 운영합니다.

요건
  • 전시/교육/시설: Front 노출용 필드(기간/장소/가격/문의/본문/이미지)가 CMS 입력 항목과 일치해야 함.
  • 신청/예약 상태값(입금확인/취소/환불/이용완료 등)은 Front 표기와 1:1 대응.
  • 휴관일/마감/정원/대기 등 운영 변수는 캘린더·신청 UX와 충돌하지 않도록 단일 정책 필요.
영화정보관리 > 영화 목록
  • 영화 목록에서 포스터/기본정보를 조회하고, 선택 삭제/등록 등의 관리 동작이 존재합니다.
  • 수동으로 데이터 갱신 기능이 별도로 표시됩니다(외부 데이터 연동 시 갱신 범위/빈도 정책 필요).
전시행사관리 > 전시/행사 목록
  • 목록/등록/수정/삭제 및 상세 정보 관리 화면이 존재합니다.
  • 등록 항목 예시: 포스터, 제목/부제, 전시·행사 기간/시간, 장소, 금액, 문의처, 비고, 본문 콘텐츠.
  • 운영: 신청자 현황(참여가 있는 경우) 진입 링크가 존재합니다.
교육관리 > 교육 목록 / 신청자 현황 / 설문 / 수료증
  • 교육 목록: 등록/수정/삭제(포스터, 제목, 한 줄 소개, 교육기간/시간, 강사, 정원, 금액, 장소 등)
  • 신청자 현황: 교육별 신청자 리스트, 예약/결제/취소 등 상태 변경, 엑셀 다운로드 등 운영 기능
  • 설문조사 관리: 문항(객관식/주관식/만족도 등)과 응답 관리(항목/필수 여부 등 세부 정책 확인 필요)
  • 수료증 관리: 수료증 발급(개별/일괄) 흐름이 포함됩니다.
시설대관관리 > 시설 목록 / 신청·예약자 현황
  • 시설 목록 예시: 공유오피스 목록, 상영관 목록 (각 시설의 소개/위치/요금/이미지 등 등록 항목 존재)
