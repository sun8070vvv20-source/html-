1. 문서 개요

요약

원본 PDF(화면설계서)의 내용을 인수인계 목적에 맞게 ‘연동·상태값·운영 개입 지점’ 중심으로 재구성한 단일 HTML 요약본입니다.

요건(문서 기반)
  • 필수원본 PDF를 최종 근거로 삼고, 본 페이지는 빠른 탐색/공유용 요약으로 사용합니다.
  • 필수커뮤니케이션은 화면ID(예: UI-MV-01-002) 기준으로 통일합니다.
  • 권장‘확인 필요/리스크’ 항목은 개발·운영 합의 후 문서/기능에 반영합니다.
  • 권장섹션 링크(#앵커)로 필요한 위치를 공유해 인수인계 비용을 줄입니다.

상세

이 페이지는 원본 PDF 화면설계서를 기반으로, 인수인계 목적에 맞게 핵심 기능/규칙/연동/상태값 중심으로 재구성한 요약본입니다. “처음 보는 사람도” 프로젝트를 따라갈 수 있도록 IA → 연동 → 공통룰 → 모듈별 상세 순서로 정리했습니다.

2. IA / 화면 체계

요약

문서는 UI-XX 화면ID 체계를 기준으로 IA(메뉴/화면)를 관리합니다. 인수인계·QA·개발 협업에서 화면ID가 공통 언어 역할을 합니다.

요건(문서 기반)
  • 필수화면ID 프리픽스(UI-JU/UI-AU/UI-MV 등)와 모듈(메뉴) 매핑을 유지합니다.
  • 필수GNB/메뉴는 영화·전시·행사·교육·시설대관·센터소개·캘린더·검색·마이페이지 축을 기준으로 구성합니다.
  • 권장원본 PDF의 ‘화면ID 목차’와 실제 구현 화면/URL의 1:1 대조표를 만들어 운영/QA 기준점을 고정합니다.
  • 권장화면(라우팅/컴포넌트)에 screen-id 메타데이터를 남겨 커뮤니케이션 비용을 줄입니다.

상세

원본은 화면ID 체계를 사용합니다(예: UI-MV-01-002). 인수인계/협업 시 “화면ID”로 커뮤니케이션하면 가장 빠릅니다.

3. 연동/외부 시스템

요약

외부 연동은 디트릭스(영화 예매), KOBIS(영화인 검색), 소셜로그인(카카오/네이버), 알림톡(운영)이 핵심 축입니다.

요건(문서 기반)
  • 필수영화 예매/확인은 디트릭스(API/솔루션) 연동 범위(영화정보·상영시간·좌석·할인·결제·취소)를 확정합니다.
  • 필수영화인 멤버십 신청/수정은 KOBIS Open API 기반 영화인 검색/선택 UI를 제공합니다.
  • 필수소셜 로그인은 ‘홈페이지 가입 여부 확인 불가’에 따른 미가입/미연동 분기 플로우를 포함합니다.
  • 권장알림톡 발송 시점·대상(어린이 교육 포함)·템플릿을 운영과 합의해 고정합니다.
  • 권장관리자(백오피스)에서 직접 관리되는 콘텐츠/미디어/정책의 데이터 소스와 권한을 명확히 정의합니다.

4. 공통 정책/룰

요약

여러 모듈에서 공통으로 쓰이는 상태값 패턴, 파일 업로드 정책, 로그인 보안 규칙을 한 번에 정리한 섹션입니다.

요건(문서 기반)
  • 필수모듈별 상태값/전이(버튼 노출 조건 포함)를 표로 확정하고 코드(Enum)와 1:1 매핑합니다.
  • 필수파일 업로드는 허용 확장자·용량(30MB)·단일 업로드 등 정책을 프론트/서버에서 동시에 검증합니다.
  • 권장보안 요구에 따라 MIME 검증·악성 파일 스캔·보관/삭제 정책을 추가 확정합니다.
  • 필수일반 로그인 5회 실패 시 확인창/문구 등 보안 정책을 실제 동작 기준으로 구현·검증합니다.

  • 소셜 로그인은 “홈페이지 가입 여부를 판단할 수 없음” 이슈로 별도 안내/연동 플로우를 둠.

5. 회원/인증

요약

회원가입(일반/어린이), 로그인(일반/소셜), 아이디·비밀번호 찾기 등 인증 전반을 다룹니다. 소셜 연동 분기와 보안 문구가 핵심입니다.

요건(문서 기반)
  • 필수일반/어린이 회원가입 플로우를 분리하고, 문서 기준 필드·본인인증 수단을 적용합니다.
  • 필수카카오/네이버 소셜 로그인과 마이페이지 내 소셜 계정 연동/해제를 제공합니다.
  • 필수로그인 실패 5회 정책 및 안내 문구(웹취약점 진단 대응 이력)를 반영합니다.
  • 권장아이디/비밀번호 찾기의 인증 수단(휴대폰/이메일) 최종안을 확정하고 예외 케이스를 정리합니다.

상세

회원가입(일반/어린이) 핵심 포인트
  • 일반/어린이 가입 플로우가 분리되어 있으며, 본인인증 수단/필드가 개정 이력에서 여러 번 변경되었습니다(이메일 ↔ 아이핀 등).
  • 회원가입 시 선호장르 정보를 활용해 영화 영역에서 “이 영화 어때요?” 추천 아이콘에 영향을 줍니다.
소셜 로그인(네이버/카카오)
  • 소셜 제공 데이터만으로 “이미 홈페이지 회원인지” 판단이 어렵다는 사유로, 미가입/미연동 확인창을 둡니다.
  • 네이버는 “네이버 식별자”로 기존 가입자 여부를 판단하는 흐름이 설계되어 있습니다.
  • 마이페이지에서 소셜 계정 연동/해제 가능(연동 시 연결 상태 표시, 해제 시 확인창).
Kakao Naver 연동/미가입 분기
아이디/비밀번호 찾기
  • 아이디/비밀번호 찾기 인증 수단이 분리(휴대폰 인증/이메일 인증), 비밀번호 찾기에서 이메일 본인인증 수단 추가 등 이력이 존재합니다.
  • 실제 구현 시 “어떤 인증수단이 최종 확정인지”는 운영/정책 문서(또는 개발 완료 버전)와 대조 필요합니다.

6. 영화 예매/확인

요약

디트릭스 연동 기반으로 영화 목록/상세/예매(좌석·할인·결제)와 예매확인을 제공합니다. 비회원 예매/확인도 포함됩니다.

요건(문서 기반)
  • 필수디트릭스 연동으로 영화정보·상영시간·좌석·할인·결제·예매확인 흐름을 제공합니다.
  • 필수회원 예매내역 + 비회원 예매 및 확인 플로우를 모두 지원합니다.
  • 필수인원/좌석/할인 선택 단계에서 제약(할인 불가 등)과 오류 안내를 명확히 노출합니다.
  • 권장선호장르 기반 ‘이 영화 어때요?’ 아이콘 노출 조건을 구현합니다.
  • 확인영화 상세의 ‘메인 예고편 영상’이 관리자 설정/불러오기인지 연동 범위를 확인합니다.

상세

기능 범위
  • 상영예매(상영중/상영예정) 목록, 상세
  • 인원/좌석 선택, 할인 선택, 결제
  • 예매내역 확인(회원), 비회원 예매 및 확인
  • 지난 상영작 목록/상세(아카이브 성격)
  • 할인혜택 안내 페이지(운영 규정 반영)
디트릭스 연동 관리자 설정(예고편)
디트릭스 연동 포인트(요약)
  • 영화 정보(제목/개봉일/장르/러닝타임 등)를 디트릭스 API로 불러오는 설계가 포함됩니다.
  • 영화 예매는 “디트릭스 영화예매 솔루션 연동”으로 명시되어 있습니다.
  • 할인 선택은 디트릭스에서 할인 종류 정보를 불러오며, 조건에 따라 “할인 적용이 불가합니다” 등의 안내가 노출됩니다.
추천/태그
  • “이 영화 어때요?” 아이콘: 회원가입 시 등록된 선호장르에 해당하는 영화일 경우 표시.
  • 상영 상태 태그 예: 상영예정(D-남은일수), 상영중, 상영마감 등.
운영/확인 포인트
  • 영화 상세에서 “메인 예고편 영상”은 관리자에서 설정한 영상 불러오기(기능 확인 필요)로 명시되어 있습니다.
  • 예매 취소 가능 시간/극장별 유의사항 등은 예매내역에서 표기합니다.

7. 전시·행사

요약

전시·행사는 게시형 콘텐츠로 목록/상세, 진행상태 태그, 비고, 공유 기능이 핵심입니다. 통합검색/캘린더 노출도 함께 고려합니다.

요건(문서 기반)
  • 필수전시·행사 목록/상세 + 진행상태 태그 + 비고 + 공유(링크 복사 등) 기능을 제공합니다.
  • 권장통합검색/캘린더 노출 필드(제목/기간/장소/썸네일/태그) 정합성을 운영 기준으로 관리합니다.
  • 권장상태 태그 워딩(예: ‘종료’) 및 표시 규칙을 운영과 합의해 고정합니다.

상세

기능 범위
  • 전시·행사 목록/상세(진행상태 태그 포함)
  • 상세 상단 비고 항목, 공유 기능(링크 복사 등) 설계가 포함됩니다.
  • 진행상태 워딩 개정 이력: “진행종료 → 종료”
운영 포인트
  • 콘텐츠(전시/행사 정보)는 관리자에서 관리되는 전형적인 게시형 구조로 보입니다.
  • 통합캘린더/통합검색에도 노출되므로, 노출 필드(제목/기간/장소/썸네일/태그) 일관성 중요.

8. 교육

요약

교육 신청은 강좌 접수 후 ‘접수 마감 뒤 결제(마이페이지)’ 구조가 특징입니다. 쿠폰, 알림톡, 취소/환불, 첨부파일 정책이 포함됩니다.

요건(문서 기반)
  • 필수교육 신청/신청완료 흐름과 ‘접수 마감 후 결제(마이페이지)’ 정책을 구현합니다.
  • 필수결제(무통장입금), 쿠폰 선택, 결제 안내 알림톡 발송(어린이 교육: 신청자+자녀)을 제공합니다.
  • 필수취소/환불(환불계좌 등록 → 환불요청 → 관리자 처리) 흐름과 상태 필터를 제공합니다.
  • 필수교육 첨부파일 업로드 정책(단일/30MB/허용 확장자/필수 여부)을 반영합니다.
  • 권장교육 유형(영화인 교육/일반 교육) 분류 및 문구/노출 규칙을 문서 기준으로 적용합니다.

상세

핵심 흐름
  1. 교육강좌 목록/상세에서 신청
  2. 신청 완료 화면(유형별 안내 문구 상이)
  3. 결제는 “접수 마감 후” 진행(마이페이지 신청내역에서 결제)
  4. 입금 확인/상태 변경(관리자 처리 + 일부 자동 변경)
  5. 필요 시 취소/환불(환불계좌 등록 후 환불요청)
무통장입금(계좌) 알림톡
유형(개정 이력 반영)
  • 기존: 성인/어린이/청소년 → 변경: 영화인 교육(영화인 대상), 일반 교육(전체/성인/어린이·청소년)
결제/쿠폰
  • 마이페이지 결제 화면에서 쿠폰 드롭다운으로 미사용 강좌 할인 쿠폰을 선택(없으면 “사용 가능한 쿠폰이 없습니다”).
  • 입금계좌: 서울영화센터 계좌 표시, 입금자명: 회원가입 시 등록된 이름.
  • 결제하기 완료 후 결제 안내 알림톡 발송(어린이 교육은 신청자+자녀 번호).
상태/취소·환불
  • 마이페이지 취소/환불내역 탭(교육/시설대관) 제공.
  • 교육 상태 필터 예시: 대기신청취소, 신청취소, 수강취소, 환불요청, 환불완료.
  • 환불요청 버튼 클릭 시 환불요청 확인창 → 환불계좌 등록 모달 등 흐름이 설계되어 있습니다.
첨부파일
  • 교육 신청/수정 시 첨부파일 업로드가 포함됩니다(형식/용량 제한 존재).

9. 시설대관

요약

시설대관은 시설별 예약 규칙(캘린더/시간/좌석)과 신청 후 입금확인 기반 상태관리가 핵심입니다. 공유오피스는 영화인 멤버십만 예약 가능으로 명시됩니다.

요건(문서 기반)
  • 필수시설 목록/상세, 예약 캘린더/시간 선택(상영관은 좌석 선택), 대관 신청(파일 첨부)을 제공합니다.
  • 필수시설 유형별 최소 예약 단위/날짜 범위 등 예약 규칙을 적용합니다.
  • 필수공유오피스 예약은 영화인 멤버십 회원만 가능하도록 권한/가드를 구현합니다.
  • 필수입금확인 프로세스(계약금/잔금 등 운영 관리 포인트 포함)를 반영합니다.
  • 확인예약폼 상세 기능(상태 색상 규칙 등) ‘확인 필요’ 항목을 실제 운영/구현 기준으로 확정합니다.

상세

기능 범위
  • 시설대관 목록/상세(시설별 안내/대관료/신청서 양식)
  • 예약 캘린더/시간 선택, (상영관은 좌석 선택 포함)
  • 대관 신청(파일 첨부), 입금/승인/완료 상태 관리
무통장입금(계좌) 기능 확인 필요(문서 표기)
예약(캘린더/시간/좌석) 핵심
  • 예약폼 상세 기능 확인(문서 내 명시).
  • 시설 유형(상영관/다목적실/회의실/공유오피스)별로 최소 예약 시간 단위 및 날짜 선택 범위가 다릅니다.
  • 공유오피스는 영화인 멤버십 회원만 예약 가능이라고 명시되어 있습니다.
  • 상영관은 좌석 선택: 이미 예약된 좌석은 선택 불가, 좌석 선택은 필수.
결제/운영
  • 시설대관 신청 후 입금 확인 프로세스가 있으며, 계약금/잔금 등은 운영에서 관리한다는 설명이 포함됩니다.
  • 시설대관과 영화인 멤버십은 “입금확인완료”까지 공통 프로세스를 공유합니다.
첨부파일(대관신청서)
  • 신청서 다운로드 후 작성 → 파일 첨부 업로드
  • 형식: hwp, pdf, doc, docx, xls, xlsx, ppt, pptx / 30MB 이하 / 단일 업로드

10. 멤버십

요약

일반 멤버십은 연간(1~12월) 영화 관람 실적을 기준으로 등급(현무/주작/청룡/백호 등)을 산정하고 혜택/쿠폰을 제공합니다.

요건(문서 기반)
  • 필수등급 산정 기준(홈페이지 예매+관람완료만 집계, 현장예매 제외)과 산정 시점(매년 1월), 유지기간(1년)을 구현합니다.
  • 확인2026년 상반기 실적으로 일괄 적용 등 예외 규칙의 적용 범위/최신 정책을 확인합니다.
  • 권장쿠폰/혜택 발급·사용 정책은 ‘운영 규정 반영’ 문서와 싱크를 맞춰 고정합니다.

상세

일반 멤버십(등급/혜택)
  • 등급 예시: 현무, 주작, 청룡, 백호
  • 등급 산정: 1년(1~12월) 영화 관람 실적 기준(홈페이지를 통해 예매 + 관람완료 건만 집계, 현장예매 제외).
  • 2026년은 예외적으로 상반기(1~6월) 관람실적으로 일괄 적용한다는 설명이 포함됩니다.
  • 등급 산정 시점: 매년 1월, 등급 유지 기간: 1년.
쿠폰/할인(운영)
  • 멤버십 혜택으로 쿠폰이 발급/노출될 수 있으며, 식음료 할인쿠폰의 경우 “매점에서 쿠폰 제시 → 직원이 할인 적용 확인 후 적용” 형태의 흐름이 설계되어 있습니다.
  • 정확한 혜택/쿠폰 종류/발급 정책은 “홈페이지 약관 운영 규정 반영” 파일의 해당 시트(문서에서 참조)로 최종 확인 필요.

11. 영화인 멤버십

요약

영화인 멤버십은 연회비(무통장입금) 기반 신청형 멤버십입니다. KOBIS 영화인 검색, 관심 영화 정보(필수), 상태값/환불 규정이 핵심입니다.

요건(문서 기반)
  • 필수연회비 60,000원 무통장입금 + 상태값(입금확인중/완료, 확정/반려 등) 처리 흐름을 제공합니다.
  • 필수KOBIS Open API로 영화인 검색/선택 UI를 제공합니다(카테고리+키워드).
  • 필수관심 영화 정보(유형/제작국가/장르) 필수 입력을 반영합니다.
  • 필수환불요청 가능 기간(입금확인완료 처리일 기준 7일) 룰을 구현합니다.
  • 필수첨부파일 업로드 정책(단일/30MB/허용 확장자)과 ‘신청/반려 상태에서 수정 가능’ 범위를 반영합니다.

상세

개요
  • 연회비 60,000원 결제(무통장입금)로 가입하는 별도 멤버십입니다.
  • 신청 시 KOBIS 영화인 검색을 통해 영화인 정보를 조회/선택합니다.
  • 관심 영화 정보(유형/제작국가/장르)를 선택합니다(필수).
KOBIS 연동 무통장입금(계좌)
상태값(대표)

문서에 언급된 상태값/흐름을 인수인계 관점으로 정리했습니다.

  • 신청 → (관리자) 입금확인중 → 입금확인완료
  • 신청확정 / 신청반려 (상태 변경 시 알림톡 발송)
  • 취소/환불요청 → 취소/환불완료
  • 유효기간만료(유효기간 1년)
환불 규정(핵심)

상세

구성
  • 서울영화센터 소개(다국어)
  • 공지사항

13. 통합캘린더

요약

요건(문서 기반)
  • 필수영화 일정은 디트릭스 API 기반으로 실시간 조회/표시합니다.
  • 권장전시·행사/교육 일정은 관리자 데이터 정합성(기간/장소/노출 필드) 검증 루틴을 마련합니다.
  • 권장타임존/변경/취소 등 이벤트에 대한 표시·캐시·장애 대응 정책을 확정합니다.

상세

무엇을 보여주나?
  • 서울영화센터에서 진행하는 영화 · 전시·행사 · 교육 일정 통합 조회
  • 영화 일정은 실시간으로 디트릭스 API에서 정보 불러옴
운영 포인트
  • 전시·행사/교육은 관리자 데이터 정합성이 중요(캘린더 노출 필드/기간/장소 등).
  • 영화는 외부 연동(디트릭스)이므로 “타임존/상영시간 변경/상영취소” 같은 이벤트 처리 전략 확인 필요.

14. 통합검색

요약

영화/전시·행사/교육/시설대관/서울영화센터(서브메뉴 포함)를 통합 검색합니다. 영화 결과의 데이터 소스는 문서에서 ‘확인 필요’로 남아 있습니다.

요건(문서 기반)
  • 필수카테고리별 통합검색 결과/필터 UX를 제공합니다(서울영화센터 서브메뉴 포함).
  • 확인영화 검색이 디트릭스 API 직접조회인지, 별도 콘텐츠 DB인지(범위/연결 포함) 확정합니다.
  • 권장검색 색인/정렬/부분일치 규칙과 운영 가이드를 문서화합니다.

상세

검색 대상(카테고리)
  • 영화
  • 전시·행사
  • 교육
  • 시설대관
  • 서울영화센터 (공지/이벤트/FAQ/Q&A 등 서브 메뉴 포함)
확인 필요
  • 문서에 “영화예매는 확인필요(디트릭스 API로 바로 불러오는지)”라는 메모가 있습니다.
  • 따라서 검색에서 ‘영화’ 결과가 “콘텐츠(영화 정보)”인지, “상영/예매 영역”까지 포함하는지, 실제 연동 범위를 확정해야 합니다.

15. 마이페이지

요약

마이페이지는 신청/결제/환불/연동 관리의 허브입니다. 교육/시설대관/영화인 멤버십 신청내역과 영화 예매내역(디트릭스)이 모입니다.

요건(문서 기반)
  • 필수신청내역의 상태 기반 버튼 노출(결제/환불요청/수료증/설문 등)을 반영합니다.
  • 필수취소/환불 내역 탭(키워드 검색+상태 필터)을 제공하고 관리자 처리 단계를 반영합니다.
  • 필수소셜 계정(카카오/네이버) 연동 상태 표시 + 연동/해제 플로우를 제공합니다.
  • 권장첨부파일 업로드/수정 가능 조건 등 예외 케이스를 문서 기반 테스트 케이스로 정리합니다.

상세

역할
  • 사용자 개인 정보/연동 관리 + 신청/결제/환불의 “업무 허브”
  • 영화 예매 내역(디트릭스 연동), 교육/시설대관 신청내역, 영화인 멤버십 신청/상태
신청내역(교육 예시)
  • 상태별로 결제하기/환불요청/수료증/설문 등의 버튼 노출이 달라집니다.
  • 신청/신청반려 상태에서만 신청정보 수정 가능(수정 시 상태값 ‘신청’으로 변경).
  • 첨부파일 업로드(단일/30MB/형식 제한) 포함.
취소/환불 내역
  • 교육/시설대관 탭 제공, 키워드 검색 및 상태 필터 제공.
  • 환불요청→환불완료는 사용자 요청 + 관리자 처리 단계가 존재합니다.
소셜 계정 연동
  • 카카오/네이버 연동 여부 표시
  • 연동/해제 시 확인창, 해제 처리 후 연동 상태 갱신

16. 확인 필요/리스크

요약

문서에 ‘확인 필요’로 표기된 항목과 인수인계 관점에서 리스크가 큰 포인트를 한 곳에 모았습니다.

요건(문서 기반)
  • 필수디트릭스/검색/시설대관 예약폼/예고편 등 ‘확인 필요’ 항목을 담당자(개발·운영)와 확정 후 반영합니다.
  • 필수상태값/환불/파일 업로드/보안 정책을 실제 구현 동작과 1:1로 점검합니다.
  • 권장API 키/쿼터/테스트 환경 등 외부 연동 체크리스트를 프로젝트 위키에 상시 업데이트합니다.

상세

문서에서 직접 “확인 필요”로 표시되거나, 인수인계 관점에서 리스크가 큰 항목
  • 시설대관 예약폼 상세 기능 확인(시설별 최소 시간 단위/날짜 범위/상태 색상 규칙 등).
  • 영화 관리자 설정 “메인 예고편 영상” 불러오기 기능 확인 필요.
  • 통합검색 영화 영역이 디트릭스 API로 “바로” 조회되는지(검색 범위/데이터소스) 확인 필요.
  • 약관/운영규정 교육/시설대관/영화인멤버십 약관 내용은 별도 파일(운영 규정 반영 시트) 참조로 되어 있어, 최종 텍스트 확정/동기화 필요.
  • 알림톡 발송 시점/템플릿/대상(어린이 교육: 신청자+자녀)을 운영/개발이 합의해야 함.
추천 체크리스트(바로 실행용)
  1. 디트릭스: API 범위(영화 정보/상영시간/좌석/할인/결제/취소), 장애 시 fallback, 테스트 환경 유무 확인
  2. KOBIS: API 키/쿼터/검색 UI(영화인명/필모), 데이터 정합성 확인
  3. 신청/결제 상태값: 교육/시설대관/영화인 멤버십 각각의 상태전이 표 확정
  4. 환불: “환불요청” 주체(사용자) + “환불완료” 주체(관리자) 및 회계 처리 흐름 확인
  5. 파일 업로드: 서버 검증(확장자/MIME/용량), 저장 위치, 보관/삭제 정책 확정
  6. 보안: 로그인 5회 실패 정책(잠금? 캡차? 확인창?)의 실제 동작 정의
